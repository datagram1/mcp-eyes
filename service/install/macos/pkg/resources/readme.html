ScreenControl - Important Information

After Installation

After installation completes, you'll need to:

  1. Grant Accessibility permissions - Required for GUI automation. The app will guide you through this process.
  2. Configure the agent - Open ScreenControl from your menu bar and enter your license key or control server URL.
Important: The ScreenControl service runs as a background process with elevated privileges. This is required for secure machine operations like screen lock/unlock.

Architecture

  • Service (/Library/PrivilegedHelperTools/com.screencontrol.service) - Handles file operations, shell commands, and machine control securely.
  • Agent (/Applications/ScreenControl.app) - Menu bar app handling GUI operations, screenshots, mouse/keyboard control.

Ports Used

  • 3459 - Service HTTP API (localhost only)
  • 3460 - GUI Bridge (service to agent communication)
  • 3457 - Browser extension WebSocket (optional)

Security

ScreenControl implements security hardening:

  • Protected paths (credentials, system files) cannot be accessed
  • Dangerous commands are blocked
  • All connections are localhost-only except for the encrypted control server connection

Uninstalling

To uninstall, run the following in Terminal:

sudo /Library/Application\ Support/ScreenControl/uninstall.sh
